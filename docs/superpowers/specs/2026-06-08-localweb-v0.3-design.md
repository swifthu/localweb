# localweb v0.3 Design — 项目源头深度识别

**Date:** 2026-06-08
**Status:** Approved (brainstorming complete)
**Author:** jimmyhu

---

## 1. Purpose

v0.2 让 label 知道了"是什么程序"(python / node / vite),但用户反馈:**看不出是哪个项目**。多个项目都用 `node` 跑 dev server,单看 label 无法区分。

v0.3 解决"用当前端口的到底是哪个项目"——通过三个方向叠加:

- **A. cwd 项目名提取** — 从工作目录路径拿最后一段作为 project name
- **B. HTTP `<title>` 探测** — dev server 自报家门(最准的来源)
- **D. 父进程链** — 展示"谁启动的"(IDE / Terminal / Docker / npm script)

UI 配套:服务卡片主标签改成"程序 · 项目"两行结构,信息密度提升一档。

## 2. Goals

- 每个服务卡片的 label 形如 `Node · myapp` 或 `Vite · dashboard-ui` —— **一眼看出"哪个项目"**
- 每个服务有 `parentChain` 字段,UI 副行展示 `↑ launchd → Terminal → zsh → npm → node`(最多 5 层)
- HTTP 服务在可能时探测 `<title>`,作为 confidence "high" 的额外 label 提示
- v0.2 行为完全保留(向后兼容,新字段全部 optional)
- 73 个现有测试全部通过

## 3. Non-Goals (本期不做)

- macOS-specific:读 Info.plist 的 CFBundleName(spec §3 已排除)
- 进程图标(macOS / Linux 平台差异大,推迟)
- 复杂的进程祖先树可视化(用一行文字链足够)
- 项目级"包管理器"识别(npm vs pnpm vs yarn)
- Windows 兼容(继续 macOS + Linux only)

## 4. 范围增量(对比 v0.2)

| 方向 | v0.2 状态 | v0.3 增量 |
|---|---|---|
| detector 命令匹配 | 11 命令行 + 7 命令名规则 | 加 cwd 提取 projectName 规则(新规则) |
| HTTP 探测 | probeHttp 读响应头 | 新 probeHttpTitle 读 HTML `<title>` |
| procinfo | exePath / startedAt / ppid | 加 getParentChain(depth) 递归 |
| Service 类型 | 16 字段 | 加 `projectName?` / `parentChain?` / `httpTitle?` 3 字段 |
| Service 卡片 | 单行 label | 主行 `label · projectName` + 副行父链 + meta |

## 6. 数据流扩展

### 6.1 启动 + 扫描(不变)

```
localweb → config → port → http → ws → preshared → scanner(2s tick)
```

### 6.2 扫描循环扩展

```
scanner.ts tick(每 2s):
  ├─ runLsof() → rawPorts
  ├─ procinfo.readExePath / readStartTime / readPpid / getParentChain(pid, 3)  ← v0.3 新增
  ├─ detector.enrich(rawPort, procinfo)
  │     ├─ 命令行匹配 → label
  │     ├─ HTTP 头探测 → label 增强
  │     ├─ cwd basename → projectName       ← v0.3 新增
  │     └─ HTTP title 探测 → httpTitle       ← v0.3 新增
  ├─ presets.lookup(port) → servicePreset
  └─ groupKey = basename(exePath) || command
```

### 6.3 项目名推断规则(detector 新增)

```
优先级(从高到低):
  1. 命令行已显式匹配 label(v0.2 的 11 个规则,如 "next dev" → "Next.js")
  2. cwd basename(从 path.basename(cwd) 提取)
     - 排除: "/" "/Users/<user>" "/usr/libexec" "/System/Library" 等系统路径
     - 排除: 路径含 ".app" (macOS app bundle,exePath 已经覆盖)
  3. 兜底:不放 projectName,UI 显示原 label
```

### 6.4 HTTP title 探测(probeHttpTitle)

```
输入: address, port
输出: { title: string, httpTitle: string } | undefined

实现:
  try:
    res = await fetch(`http://${address}:${port}/`, { signal: AbortController(1s) })
    html = await res.text()
    match = html.match(/<title[^>]*>([^<]+)<\/title>/i)
    if match: title = match[1].trim()
  catch: return undefined

按 (address, port) 缓存(procinfo 的 1s 缓存同理)
```

## 7. 类型扩展

```typescript
// types.ts: Service 新增 3 字段(全部 optional)
interface Service {
  // ... 现有 16 字段(v0.1 + v0.2)

  // v0.3 新增
  projectName?: string;     // cwd basename,如 "myapp"
  parentChain?: string;     // "launchd → Terminal → zsh → npm → node"
  httpTitle?: string;       // 探测到的 HTML <title>,如 "Vite + React"
}
```

## 8. UI 设计

### 8.1 服务卡片(新)

```
┌─────────────────────────────────────────────────────────────────────┐
│ Node · myapp                              [Open] [Copy URL] [Kill]   │  ← 主行:label + projectName
│ pid 12345 · npm run dev · 127.0.0.1:3000 · started 2h ago            │  ← 副行:meta
│ /Users/jimmyhu/.nvm/versions/node/v26/bin/node                       │  ← exePath
│ ↑ launchd → Terminal → zsh → npm → node                            │  ← 父链(独立行,indigo 色)
│ Title (探测到时): "Vite + React - myapp"                            │  ← httpTitle(独立行)
└─────────────────────────────────────────────────────────────────────┘
```

### 8.2 视觉规则

- 主行: `label · projectName` (· 用 middle dot)
- 父链行: `↑` 开头,`var(--accent)` 颜色
- httpTitle 行: 只在探测到时显示,muted 颜色,前缀 "Title:"
- 系统服务(无 projectName,parentChain 只有 launchd):不显示项目名/父链行,保持原貌

## 9. 错误处理

| 场景 | 处理 |
|---|---|
| cwd 是 "/" 或 "/usr/libexec" | 不生成 projectName(系统进程) |
| HTTP 探测超时 1s | httpTitle = undefined,降级 |
| HTTP 探测返回非 HTML | 不解析 title,降级 |
| getParentChain 死循环(pid → ppid 循环引用) | depth 限制 5 层 + Set 检测 |
| 父链中 pid 已死 | readCommand(pid) 返回 undefined,该节点显示 "?" |

## 13. 风险与权衡

| 风险 | 缓解 |
|---|---|
| HTTP 探测每个服务 +1 fetch,可能 25+ 并发请求 | 1s 超时 + 按 (address, port) 缓存,扫描期不重复 |
| 父链递归可能误判(zombie process / 容器 PID 1) | depth 5 限制 + 已死 PID 显示 "?" |
| cwd basename 在某些 monorepo 不可靠(根目录运行) | 退回命令行路径提取(plan §6.3 没做,v0.4 再说) |
| macOS 系统服务(cfprefsd, rapportd 等) cwd 不可读 | readlink /proc 时 catch,exePath 仍可填 |
| Service 字段数从 16 涨到 19,JSON 体积涨 ~30% | 25 个服务 ~3KB 增长,可忽略 |
